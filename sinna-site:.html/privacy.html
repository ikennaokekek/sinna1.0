<h1>Privacy Policy</h1>

<p><strong>Last updated:</strong> 2025-09-12</p>

<h2>Who we are</h2>

<p>Sinna 1.0 is an accessibility API for streaming services. Contact: <a href="mailto:support@sinna.app">support@sinna.app</a>.</p>

<h2>What we process</h2>

<ul>

  <li><strong>Media you send:</strong> video/audio files or URLs for captioning and audio description.</li>

  <li><strong>Derived data:</strong> transcripts, captions/subtitles (VTT/SRT/TTML), audio description tracks, quality reports.</li>

  <li><strong>Account data:</strong> tenant name, admin email, API keys, billing identifiers.</li>

  <li><strong>Usage data:</strong> job IDs, durations, processing metrics, error logs.</li>

  <li><strong>Payment data:</strong> handled by Stripe; we do not store card numbers.</li>

  <li><strong>Cookies:</strong> minimal session and analytics cookies on our docs/status pages.</li>

</ul>

<h2>How we use data</h2>

<ul>

  <li>Provide the service: generate captions, audio description, and export files.</li>

  <li>Operate and secure the platform: rate limiting, fraud prevention, abuse detection.</li>

  <li>Billing and support: subscription management, incident response.</li>

  <li>Product improvement: aggregated, anonymized metrics and error diagnostics.</li>

</ul>

<h2>Where data is stored</h2>

<p>Artifacts and backups are stored with reputable cloud providers. Tenants may request EU or US data residency for stored artifacts where supported.</p>

<h2>Retention</h2>

<ul>

  <li><strong>Generated artifacts</strong> (captions, AD audio, export zips): kept by default up to <strong>90 days</strong> or per-tenant policy, then deleted.</li>

  <li><strong>Job metadata and logs:</strong> up to <strong>90 days</strong> for operations and security.</li>

  <li><strong>Backups:</strong> rolling backups may persist up to <strong>30 days</strong> after deletion for disaster recovery.</li>

  <li>You may request earlier deletion; see “Your rights.”</li>

</ul>

<h2>Sharing</h2>

<ul>

  <li><strong>Processors:</strong> cloud storage/CDN, message queues, observability, and payment (Stripe) to run the service.</li>

  <li><strong>No sale of personal data.</strong></li>

  <li>We disclose if required by law or to protect the service from abuse.</li>

</ul>

<h2>Security</h2>

<ul>

  <li>TLS in transit; signed URLs for downloads.</li>

  <li>Access controls, API keys per tenant, audit logs, and key rotation.</li>

</ul>

<h2>Your rights</h2>

<ul>

  <li>Access, correct, or delete your data where applicable.</li>

  <li>Request export or deletion via <a href="mailto:support@sinna.app">support@sinna.app</a>.</li>

  <li>Configure retention and residency at the tenant level where available.</li>

</ul>
